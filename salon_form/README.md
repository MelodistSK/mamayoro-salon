# ままよろサロン 無料相談フォーム（Webフォーム → スプレッドシート集約）

talk（点で止めない。線にする。）のCTA・QRの差し込み先。回答は Google スプレッドシートに1行ずつ溜まる。
将来の「ままよろサロン管理プラットフォーム」を見据え、列構成は **開催回 × 参加者 × 回答** の最小スキーマにしてある。

## ファイル
- `index.html` … フォーム本体（自己完結・モバイル/QR前提・navy×goldのブランド）
- `Code.gs` … スプレッドシートに貼る Apps Script（受け取って行追加）

## セットアップ（10分）
1. Google スプレッドシートを新規作成（名前例：`ままよろサロン_申込`）。
2. スプレッドシートで **拡張機能 → Apps Script**。中身を全消し → `Code.gs` を貼り付け → 保存。
3. **デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**。
   - 「次のユーザーとして実行：自分」
   - 「アクセスできるユーザー：**全員**」
   - デプロイ → 初回は承認（自分のGoogleアカウントで許可）。
4. 表示された **ウェブアプリのURL** をコピー。
5. `index.html` を開き、`var ENDPOINT = "";` の `""` の中にURLを貼る → 保存。
6. `index.html` をネット上に公開（おすすめ：Netlify Drop へドラッグ＆ドロップ / GitHub Pages / Cloudflare Pages / 自社サーバー）。公開URLができる。
7. 公開URLの **QRコード** を作成し、talk の CTA に差し込む。
8. テスト送信 → スプレッドシートに「回答」シートができ、1行入ればOK。

> 送信は `mode:'no-cors'` で行う（ブラウザのCORS制約を避けるため）。フォームは送信後すぐ完了画面を表示し、記録はApps Script側でシートに残る。厳密な成否表示が必要なら iframe 方式へ変更可。

## 毎月の使い回し（定期開催）
`index.html` 上部の隠しフィールドを、その回に書き換えるだけ：
```html
<input type="hidden" name="event_id" value="MMYS-2026-08">
<input type="hidden" name="event_ym" value="2026-08">
```
回答は同じシートに「開催回タグ付き」で溜まる → 月をまたいで集計できる。

## 二段構え（ホット / ソフト）
フォーム上部で希望を切り替える。1つのフォーム・1つのシートに集約され、`種別` 列で区別できる。
- **無料相談**（ホット）… 本気の見込み。フル項目（会社名・興味の方向まで必須）。
- **案内・資料希望**（ソフト）… 温かい見込み。お名前＋メールのみ必須（会社名・以降は任意/非表示）。
将来プラットフォームでは `種別` でリードのホット/ウォームを分け、フォロー導線を変えられる。

## 収集される列（スキーマ）
`回答ID / 受付日時 / 開催ID / 開催年月 / シリーズ / 種別 / お名前 / 会社名 / メール / 電話 / 業種 / 会社規模 / 興味の方向 / 困りごと / source / 同意`
（ソフト申込では 電話・興味の方向・困りごと・業種・会社規模 は空欄になる）

## 将来プラットフォームへの移行
- この列構成 = 「開催回」「参加者（メールで名寄せ）」「回答」の3エンティティを1枚に平たくしたもの。
- kintone等へ移す時は、同じ列を3アプリに分けるだけ。フォームの `ENDPOINT` をプラットフォームのAPI/Webhook（またはLIFF経由）に差し替えれば、`index.html` はほぼそのまま流用できる。
- 「同じ場所からフォームを作り、参加者・回答が集約される」運用の最初の1ピースがこのフォーム。
